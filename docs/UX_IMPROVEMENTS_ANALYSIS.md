# Анализ сайта FreeTips: стиль, эффективность, дизайн, удобство

Отчёт по результатам аудита лендинга и ключевых компонентов.

---

## 1. Стиль и визуальная согласованность

### Что хорошо
- Единая палитра (primary/accent), CSS-переменные, Tailwind.
- Одинаковая структура секций: заголовок + подзаголовок + контент.
- Консистентные отступы (py-16 sm:py-24), max-w-6xl, скругления (rounded-2xl).

### Рекомендации

| Проблема | Решение |
|----------|--------|
| **Header всегда тёмный** (slate-900), лендинг поддерживает light/dark в классах, но тема жёстко `dark` в layout | Либо убрать неиспользуемые light-классы с лендинга (упростить), либо добавить переключатель темы и использовать `dark:` осознанно. Сейчас часть компонентов дублирует light-варианты без эффекта. |
| **Разные оттенки фона секций** (slate-50/80, white, gradient-to-b) | Унифицировать 2–3 варианта: «светлая секция», «акцентная», «нейтральная» — чтобы чередование было предсказуемым. |
| **Анимации** (animate-float, animationDelay) на многих карточках | Оставить только для Hero и, при желании, CTA. В списках (About, Adapt, ForWho) убрать или заменить на один общий fade-in при появлении в viewport (снижает визуальный шум). |
| **Иконки** в карточках — только градиентный круг | Добавить лёгкую вариацию: например, accent для 2–3 пунктов в блоке «Адаптация», чтобы визуально разбить монотонность. |

## 2. Эффективность и производительность

### Что хорошо
- Next.js App Router, standalone output.
- Нет тяжёлых внешних изображений на главной (после удаления Unsplash).
- `prefers-reduced-motion` учтён в globals.css.

### Рекомендации

| Проблема | Решение |
|----------|--------|
| **Шрифт Google Fonts** загружается по ссылке в `<head>` | Подключить через `next/font` (e.g. `next/font/google` — Plus Jakarta Sans) для самхоста и исключения блокирующего запроса. |
| **Header** при каждом переходе запрашивает `/api/profile` по pathname | Рассмотреть короткий кэш (например, 60 s) или проверку только при фокусе окна / после login/logout, чтобы не гонять запрос на каждый клик по навигации. |
| **Лендинг полностью серверный** | Оставить как есть; тяжёлые клиентские компоненты только там, где нужно (FAQ аккордеон, CookieConsent, формы). |
| **Конфиг site.ts** | Если блок `images` не используется — удалить для уменьшения размера бандла и ясности конфига. |

## 3. Дизайн и структура контента

### Что хорошо
- Понятная иерархия: один H1 на главной, логичные H2 в секциях.
- Ясный CTA: «Подключиться» / «Оставить заявку» / «Войти».
- Секции с якорями (#about, #how, #ad, #for, #quest, #contacts) — удобно для навигации и ссылок.

### Рекомендации

| Проблема | Решение |
|----------|--------|
| **Hero** справа только две карточки (Apple Pay / карта) | Добавить короткую подпись типа «Без приложения для гостя» или один социальный доказательство (число пользователей / заведений), чтобы правая колонка не казалась пустоватой на больших экранах. |
| **Блок «Адаптация»** — 6 карточек в 3 колонки | На мобильных 2 колонки — ок. Для сканирования можно добавить подзаголовки-категории (например, «Подключение», «Настройки», «Финансы») или визуально сгруппировать. |
| **Контакты** в подвале и секция «Контакты» | Дублирование email/телеграм. В секции «Контакты» можно добавить телефон (если есть в site.footer.support) и кнопку «Позвонить» для мобильных. |
| **Footer** три колонки на десктопе | На узких экранах порядок: сначала документы и поддержка, потом реквизиты — проверить логический порядок для мобильных. |
| **Название сайта** | В config — «FreeTips», в metadata — «Чаевые картой мгновенно». Для SEO и бренда: вынести название продукта в metadata (title/description) согласованно. |

## 4. Удобство (UX) и доступность

### Что хорошо
- Skip link «Перейти к основному содержимому», main с id и tabIndex.
- Кнопки и ссылки с min-h 44–48px, focus-visible кольца.
- Семантика: header, main, footer, nav, section, aria-label где нужно.

### Рекомендации

| Проблема | Решение |
|----------|--------|
| **CookieConsent** кнопка «Понятно» | Задать min-h-[44px] и focus-visible, как у остальных интерактивных элементов. |
| **ContactsSection** | Ссылки mailto/telegram — добавить min-h для зоны нажатия и focus-visible. |
| **Мобильное меню** | Закрытие по клику на оверлей есть; убедиться, что при открытом меню фокус не уходит «за» модал (focus trap) и возврат фокуса на кнопку «Меню» после закрытия. |
| **Формы** (логин, заявка, регистрация) | Ярлыки полей связать с input через id/for; единый стиль сообщений об ошибках (цвет, иконка, место под полем). |
| **Загрузка страниц кабинета/админки** | Вместо пустого экрана показывать скелетон или спиннер в контейнере контента (уже есть скелетоны в части страниц — унифицировать). |

## 5. Мелкие улучшения

- **Логотип в Header**: у `<img>` указать `alt={site.logo.alt}` (сейчас в Link есть aria-label, но у img alt пустой в Footer — в Header alt есть).
- **Footer**: у логотипа img не должно быть `alt=""` при наличии текста рядом — можно оставить alt для логотипа для скринридеров.
- **Язык**: на всех страницах `lang="ru"` в layout — ок.
- **Адаптив**: проверка вьюпортов 320px, 768px, 1024px — сетки и переносы кнопок (flex-wrap в Hero) выглядят корректно.

## 6. Приоритизация

| Приоритет | Действие |
|-----------|----------|
| **Высокий** | next/font для шрифта; унификация загрузки (скелетоны) в кабинете/админке; min-h и focus у CookieConsent и контактных ссылок. |
| **Средний** | Убрать неиспользуемый `images` из config (если не используется); уменьшить количество анимаций на лендинге; добавить телефон в секцию «Контакты» при наличии. |
| **Низкий** | Небольшие правки стиля (чередование фонов секций, акцентные иконки в Adapt); опционально — переключатель темы и использование light-режима. |
