# Подключение ФЛ и ЮЛ: критерии и проверка

**Цель:** зафиксировать требования к заявителям (физлицо и юрлицо), данные для проверки и чек-лист суперадмина при одобрении заявки.  
**Связь:** форма заявки (`/zayavka`), админка заявок, [ESTABLISHMENT_IDEA_AND_OPTIONS.md](./ESTABLISHMENT_IDEA_AND_OPTIONS.md) (раздел 9).

---

## 1. Тип заявителя

| Тип | Код | Описание |
|-----|-----|----------|
| **Физическое лицо (ФЛ)** | `FL` | Подключение как получатель чаевых (официант, курьер, мастер и т.п.) без привязки к юрлицу/заведению или с последующей привязкой к заведению как официант. |
| **Юридическое лицо (ЮЛ)** | `UL` | Подключение от имени организации (кафе, ресторан, доставка). Создаётся заведение, назначается управляющий; официанты подключаются уже через кабинет заведения. |

В форме заявки первый шаг — выбор «Подключаюсь как физическое лицо» / «Подключаюсь от имени организации (юрлицо)». Дальше показываются поля в зависимости от выбора.

---

## 2. Требования к подключению ФЛ

| № | Требование | Как проверяем | Поле в заявке |
|---|------------|---------------|----------------|
| 1 | Возраст не менее 18 лет (или по правилам платформы) | По дате рождения | `dateOfBirth` |
| 2 | Резидент РФ (для выплат и договоров) | По умолчанию для российского сервиса; при расширении — явный вопрос | Опционально: «Резидент РФ» (да/нет) |
| 3 | Контакт для связи и отправки ссылки на регистрацию | Email обязателен | `email` |
| 4 | Телефон для связи/верификации | Нормализованный номер | `phone` |
| 5 | Вид деятельности (для понимания контекста) | Текстом | `activityType` |
| 6 | (Опционально) Самозанятость / ИНН физлица | Для проверки через ФНС перед выплатой — на этапе выплаты, не обязательно в заявке | При желании: `inn` (ФЛ) |

**Итог для формы ФЛ:** ФИО, дата рождения, телефон, email, вид деятельности, название заведения (опционально, текст). ИНН физлица — по желанию продукта (для последующей проверки самозанятого).

---

## 3. Требования к подключению ЮЛ

| № | Требование | Как проверяем | Поле в заявке |
|---|------------|---------------|----------------|
| 1 | Организация зарегистрирована в РФ | ИНН, ОГРН/ОГРНИП — проверка через API ФНС/ЕГРЮЛ (опционально при отправке или при одобрении) | `companyName`, `inn`, `ogrn` |
| 2 | Контактное лицо (ответственный за кабинет заведения) | ФИО, должность | `contactFullName`, `contactPosition` (опционально) |
| 3 | Email и телефон для связи и регистрации управляющего | Обязательно | `email`, `phone` |
| 4 | Наименование заведения (для отображения в системе) | Текстом, можно совпадать с юрлицом или быть торговым названием | `establishmentName` или `companyName` |
| 5 | Адрес / вид деятельности | Для контекста и отчётности | `address`, `activityType` |

**Итог для формы ЮЛ:** тип заявки = ЮЛ; название организации, ИНН, ОГРН; контактное лицо (ФИО), email, телефон; название заведения (если отличается от юрлица); опционально адрес, вид деятельности.

**Проверка ЮЛ:** при вводе ИНН/ОГРН можно вызывать API ФНС (ЕГРЮЛ/ЕГРНИП) и подставлять название, статус организации, чтобы суперадмин видел актуальные данные и не подключал ликвидированные/недействующие организации.

---

## 4. Как мы проверяем

| Тип | Действие | Инструмент |
|-----|----------|------------|
| **ФЛ** | Проверка возраста по дате рождения | Расчёт по `dateOfBirth`. |
| **ФЛ** | Проверка самозанятого перед выплатой (опционально) | API ФНС (статус самозанятого); не обязательно на этапе заявки. |
| **ЮЛ** | Проверка организации по ИНН/ОГРН | API ФНС (ЕГРЮЛ/ЕГРНИП): статус (действующая/ликвидирована), название, ОГРН. В форме или в админке — кнопка «Проверить по ИНН» или авто-проверка при вводе. |
| **Общее** | Связь с заявителем | Отправка ссылки на регистрацию на указанный email после одобрения. |

Критерии «подключаем или нет» (например, не подключаем организации в процессе ликвидации) зафиксировать в правилах модерации и отразить в чек-листе ниже.

## 5. Чек-лист для суперадмина при одобрении заявки

Перед выдачей ссылки на регистрацию суперадмин проверяет по чек-листу. В интерфейсе админки можно показывать этот список и отмечать пункты.

### Заявка от ФЛ

- [ ] ФИО заполнено, дата рождения указана; возраст ≥ 18 лет (если это требование).
- [ ] Email и телефон валидны, контакты рабочие (по необходимости — звонок/письмо).
- [ ] Вид деятельности указан; заявка не вызывает сомнений (спам, тест).
- [ ] Решение: одобрить → выдать ссылку на регистрацию; отклонить → указать причину.

### Заявка от ЮЛ

- [ ] Название организации, ИНН, ОГРН заполнены.
- [ ] Проверка по ИНН/ОГРН выполнена (ЕГРЮЛ/ЕГРНИП): организация действующая, данные совпадают.
- [ ] Контактное лицо и email/телефон указаны; контакт релевантен (владелец, управляющий, ответственный).
- [ ] Название заведения для отображения задано (или = название юрлица).
- [ ] Решение: одобрить → создать заведение (Establishment), выдать ссылку на регистрацию управляющего, привязать пользователя к заведению с ролью ESTABLISHMENT_ADMIN; отклонить → указать причину.

## 6. Поля формы заявки (сводка)

Использовать при доработке формы `/zayavka` и API `POST /api/registration-requests`.

### Общие поля (ФЛ и ЮЛ)

| Поле | Тип | Обязательное | Примечание |
|------|-----|---------------|------------|
| applicantType | `FL` \| `UL` | да | Выбор на первом шаге. |
| email | string | да | Для отправки ссылки. |
| phone | string | да | Нормализация +7… |

### Только для ФЛ

| Поле | Тип | Обязательное | Примечание |
|------|-----|---------------|------------|
| fullName | string | да | ФИО. |
| dateOfBirth | string | да | Формат YYYY-MM-DD. |
| activityType | string | да | Вид деятельности. |
| establishment | string | нет | Название заведения (текст), если уже известно. |

### Только для ЮЛ

| Поле | Тип | Обязательное | Примечание |
|------|-----|---------------|------------|
| companyName | string | да | Наименование организации. |
| inn | string | да | ИНН 10 или 12 цифр. |
| ogrn | string | да (для ЮЛ) / нет (ИП) | ОГРН/ОГРНИП. |
| contactFullName | string | да | ФИО контактного лица. |
| contactPosition | string | нет | Должность. |
| establishmentName | string | нет | Торговое название заведения (если отличается от companyName). |
| address | string | нет | Адрес. |
| activityType | string | нет | Вид деятельности. |

В БД (RegistrationRequest) хранить `applicantType`; для ЮЛ добавить поля `companyName`, `inn`, `ogrn`, `contactFullName` (и при необходимости остальные) или вынести в JSON `payload` до миграции.
