# Тестовые сценарии для СБП-выплат (из Оглавление1, стр. 122–123)

Для тестирования вызовите **webapi/sbp/SBPCreditPrecheck** с указанными ниже `phone` и `recipientBankId`, затем **webapi/sbp/SBPCredit** с полученным `precheck_id`.

**Важно:** сценарии работают только на **тестовом стенде** (test.paygine.com). На боевом не использовать.

## Таблица 56. Тестовые сценарии для SBPCreditPrecheck и SBPCredit

| Название сценария | phone | recipientBankId |
|-------------------|-------|-----------------|
| **Успешный пречек и успешная СБП-выплата** | Любые значения, **не** из строк ниже | Любые, **не** из строк ниже |
| Успешный пречек, выплата в статусе REJECTED | 79110123456 | 100000000003 |
| Успешный пречек, выплата в статусе ERROR | — | 100000000004 |
| Успешный пречек, выплата в статусе TIMEOUT | — | 100000000005 |
| Успешный пречек, выплата PENDING→APPROVED | — | 100000000006 |
| Успешный пречек, выплата PENDING→REJECTED | — | 100000000007 |
| Неуспешный пречек, ошибка 281 | — | 100000000008 |
| Неуспешный пречек, ошибка 282 | — | 100000000009 |
| Неуспешный пречек, ошибка 283 | — | 100000000010 |
| Неуспешный пречек, ошибка 284 | — | 100000000011 |
| Неуспешный пречек, ошибка 285 | — | 100000000012 |

## Пример из документа (успешный сценарий)

В разделе Таблица 53 в примере запроса используются:
- **phone:** 70130610091  
- **recipientBankId:** 100000000091  

Эта пара не входит в «особые» строки таблицы 56, поэтому даёт **успешный пречек и успешную выплату**.

В коде по умолчанию: `PAYGINE_PAYOUT_DEFAULT_PHONE` / `PAYGINE_PAYOUT_DEFAULT_BANK_ID`; если не заданы — подставляются 70115110123 и 100000000091. Для воспроизведения сценария из документа можно в env задать `PAYGINE_PAYOUT_DEFAULT_PHONE=70130610091`.
