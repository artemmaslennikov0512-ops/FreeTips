# Настройка почты Mail.ru для восстановления пароля

## Часть 1: Настройки в Mail.ru

### Шаг 1. Войдите в почту
Откройте [mail.ru](https://mail.ru) и войдите в ящик, с которого будут отправляться письма (например, `tippingserviceft@mail.ru`).

### Шаг 2. Откройте настройки безопасности
1. Нажмите на ваш email (правый верхний угол) → **Настройки**.
2. Перейдите в раздел **«Пароль и безопасность»** (или **«Безопасность»**).

### Шаг 3. Включите доступ по SMTP/IMAP (если есть пункт)
Если в настройках есть пункт вроде **«Доступ к почте по протоколу IMAP»** или **«Почтовые программы»** — включите его (разрешить доступ по SMTP).

### Шаг 4. Создайте пароль приложения
1. В разделе **«Пароль и безопасность»** найдите блок **«Пароли для внешних приложений»** (или **«Пароль приложения»**).
2. Нажмите **«Добавить»** / **«Создать пароль приложения»**.
3. Укажите название (например: **FreeTips** или **Сброс пароля**).
4. Скопируйте выданный **пароль** (обычно 16–20 символов без пробелов).  
   Сохраните его — второй раз Mail.ru его не покажет.  
   Этот пароль вы подставите в `.env` в `SMTP_PASS`.

### Шаг 5. Двухфакторная аутентификация (2FA)
Если пароль приложения доступен только при включённой 2FA:
- Включите **двухфакторную аутентификацию** в настройках безопасности.
- После этого станет доступно создание пароля приложения (шаг 4).

## Часть 2: Настройки в проекте (.env)

### Шаг 6. Добавьте переменные в `.env`

В корне проекта откройте файл **`.env`** и добавьте (или раскомментируйте и заполните) строки:

```env
# Письма через Mail.ru
SMTP_HOST=smtp.mail.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=tippingserviceft@mail.ru
SMTP_PASS=сюда_вставьте_пароль_приложения_из_шага_4
SMTP_FROM=FreeTips <tippingserviceft@mail.ru>
```

Замените:
- **`SMTP_USER`** — ваш полный email на Mail.ru (логин).
- **`SMTP_PASS`** — пароль приложения из шага 4 (без пробелов и кавычек).
- **`SMTP_FROM`** — тот же ящик в формате `Имя <email@mail.ru>`.

Важно:
- Не ставьте кавычки вокруг пароля в `.env`.
- Если используете Mail.ru, не задавайте `RESEND_API_KEY` — иначе приложение может использовать Resend вместо SMTP.

### Шаг 7. Перезапустите приложение
После сохранения `.env` перезапустите сервер (например, `npm run dev` или перезапуск процесса на сервере), чтобы переменные подхватились.

## Проверка

1. Откройте сайт → **Вход** → **Забыли пароль?**
2. Введите логин или email пользователя, у которого в профиле указан email.
3. Нажмите **«Отправить ссылку»**.
4. Проверьте почту (и папку «Спам») — должно прийти письмо со ссылкой на сброс пароля.

Если письмо не приходит — проверьте логи сервера (ошибки отправки логируются). Убедитесь, что в `.env` нет опечаток и что `SMTP_PASS` — именно пароль приложения, а не основной пароль от ящика.

## Отправка токена из ЛК админа

Та же настройка почты используется при **«Выслать токен»** по заявке на регистрацию (админка → Заявки на подключение → Принять → Выслать токен на email). Если появляется сообщение *«Не удалось отправить письмо. Проверьте настройки почты на сервере»*:

1. **Проверьте логи приложения на сервере** — реальная причина ошибки (SMTP/Resend) пишется в лог с тегом `admin.registration_request.send_token.failed`.
2. **Убедитесь, что на сервере заданы те же переменные** `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` (или `RESEND_*`), и что процесс перезапущен после изменения `.env`.
3. **Проверьте исходящие подключения** — порт 465 (SMTP) должен быть доступен с сервера до `smtp.mail.ru`; при использовании VPS иногда блокируют исходящий SMTP.
4. **Mail.ru**: используйте только пароль приложения (не основной пароль); при блокировке с IP сервера можно попробовать Resend (см. комментарии в `.env`).
