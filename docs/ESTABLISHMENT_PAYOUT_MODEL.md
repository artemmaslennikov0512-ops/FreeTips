# Модель распределения чаевых в заведении

## Идея

1. **Общие чаевые** (по QR сотрудников или ручной ввод в пул) попадают в **пул заведения**.
2. **Администратор задаёт правила:**
   - **Доля заведения** — процент, который остаётся у заведения (например 10–20%). Остаток идёт в распределение между командой.
   - Опционально: **благотворительность** — X% в фонд (правило типа `charity`).
3. **Остаток пула** распределяется между **активными сотрудниками по коэффициентам** (поле `Employee.coefficient`). Например коэффициенты 1, 1, 2 → доли 25%, 25%, 50%.

## Реализация в БД

- **PayoutRule**: типы `establishment_share` (процент заведению), `charity` (процент в фонд). Для доли заведения обычно одно правило на заведение с `type: "establishment_share"` и `value: 15` (15%).
- **Распределение между сотрудниками** — не отдельные правила, а коэффициенты у каждого Employee. Формула: доля сотрудника = его coefficient / сумма coefficients всех активных сотрудников.

## Поток (будущее)

1. За период собираются чаевые по заведению (транзакции по TipLink с employeeId → пул).
2. Вычисляется доля заведения по правилу `establishment_share`.
3. Остаток распределяется по коэффициентам между сотрудниками.
4. Результат можно фиксировать в таблице начислений (Allocation) для истории и выплат официантам.
